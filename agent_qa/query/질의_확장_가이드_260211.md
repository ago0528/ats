# 이동 에이전트 질의 확장 가이드

## 📌 프로젝트 개요

### 목적
- ATS '이동 에이전트'의 품질 검증을 위한 질의 데이터셋 구축
- 기능당 2-3개의 질의를 **5개 수준**으로 확장
- 자연스러운 사용자 발화 패턴 반영

### 배경
- **제품**: 10년 이상 된 ATS(채용 관리 시스템)
- **복잡도**: 거의 모든 채용 시나리오를 커버할 수 있는 고도로 복잡한 기능
- **주 시나리오**: 싱글턴 대화 (한 번의 질의로 원하는 기능에 바로 이동)

---

## 👥 사용자 페르소나

### 주 사용자
- **역할**: HR 담당자 / 채용 담당자
- **특징**:
  - 제품을 잘 사용하지 못하는 경우가 많음
  - 기능이 복잡하고 어려워서 원하는 기능을 찾기 힘들어함
  - 급하게 처리해야 하는 상황이 빈번함

### 사용 패턴
- 채용명/전형명을 명시하지 않고 질의하는 경우 존재
  - → 시스템은 '채용 선택 테이블' 제공
  - → 사용자가 선택하면 특정 파라미터가 포함된 URL로 바로 이동
  - → 해당 기능에 자동 포커스

## 📚 용어 정리 (Glossary)

### 1. 채용 (Recruitment Plan)
- **의미**: 일종의 '채용 계획' 그룹
- **구성**:
  - 여러 개의 공고 (지원자 접수 전형)
  - 여러 개의 전형 (역량검사, 서류, 면접, 개발자 면접 등)

### 2. 지원서 관련 용어
#### 지원서 동의서
- 지원자가 개인정보 제공에 동의하는 등 채용 활동에 필요한 각종 동의서

#### 지원자 서약서
- 지원자가 최종 제출 시, 본인의 입력 사항에 허위사실이 없음을 서약하는 확인(동의)서

### 3. 중복 지원 차단
- 같은 채용 내에서 지원을 막는 기능
- 구인 기업에 N개월 내 지원한 사람의 지원을 막는 기능
- 다양한 조건 설정 가능

### 4. 지원서 작성 버튼 제공
- **숨김 설정 시**: 채용 공고 페이지는 보이지만, '지원서 작성 버튼'이 노출되지 않음
- **용도**: 공고만 보여주고 싶거나, 내부 추천만 받고 싶을 때

### 5. 평가 관련 용어
#### 최종 평가척도
- S/A/B/C 등급 또는 사용자 커스텀 평가 등급
- 리커트 척도(11점 등) 등 자유롭게 설정 가능

#### 평가점수 입력 방식
- 종합평가 점수를 입력할지, 역량평가별 점수를 입력할지 선택

### 6. 지원서 블라인드
- **위치**: 서류전형, 면접전형 (지원자 접수에는 없음)
- **항목**: 이름, 학교, 사진, 나이, 성별 등 모든 항목 블라인드 가능
- **목적**: 인사담당자 외 사람이 지원서를 볼 때 편향 방지

### 7. 요약 리포트 (서류 전형)
- '지원서 요약 리포트' 화면에 표시될 항목을 설정하는 기능
- 서류 결과 대시보드 커스터마이징

### 8. 역량검사 관련
#### 역량검사 타입
- 사용자가 커스터마이징한 역량 검사 기준 확인 기능

#### 응시자 세부 설정
- 역량검사 응시자 사이트에 노출되는 문구 설정
- 응시 시작 문구, 응시 종료 문구, 외부 링크(버튼) 추가 가능

#### 대표 구성원 vs 유사 구성원
- **대표 구성원**: 해당 역량/직무에서 '대표'가 되는 구성원
- **유사 구성원**: 지원자와 유사한 구성원
- 각각 역량검사 결과표의 다른 영역에 표시됨

#### AI 콘텐츠
- 채용 담당자/평가자가 면접 시 활용할 수 있는 가이드
- AI가 생성할 때 참고하는 기준을 설정하는 기능

### 9. 면접 전형 관련
#### 수험표
- 사용 설정 시: 구직자가 출력할 수험표를 직접 설정 가능
- HTML 에디터 제공 → QR 코드, 이미지 등 자유롭게 추가 가능

#### 면접확인서
- 구직자가 면접 참여 여부를 출력할 수 있게 하는 기능
- 면접 '후' 발급하는 증빙서류

## 📋 질의 확장 패턴 (9가지)

### 1. 기존
- 원본 CSV에 있던 질의
- 예: `"이전에 했던 채용을 복사해서 새로 만들고 싶어"`

### 2. 확장-작업목적
- 구체적인 작업 의도 표현
- 패턴: `"~하고 싶어"`, `"~할게"`
- 예: `"작년 신입 채용 그대로 다시 쓰고 싶어"`

### 3. 확장-탐색
- 기능 위치 찾기
- 패턴: `"어디서 해?"`, `"어디 있어?"`
- 예: `"과거 채용 복제하는 거 어디서 해?"`

### 4. 확장-직접실행
- 즉시 실행 의도
- 패턴: `"~보여줘"`, `"~할래"`
- 예: `"이전 채용 템플릿 가져올게"`

### 5. 확장-기능찾기
- 기능을 모르고 찾는 경우
- 패턴: `"~하는 거 어디야?"`, `"~어떻게 해?"`
- 예: `"채용 복사 기능 어디 있어?"`

### 6. 확장-문제상황
- 문제 해결 상황
- 패턴: `"~어디 갔지?"`, `"~안 돼"`
- 예: `"채용 설정 다 끝내지 못했는데 어디 갔지?"`

### 7. 확장-확인
- 상태 확인
- 패턴: `"~확인하고 싶어"`, `"~됐나 봐야지"`
- 예: `"누구한테 메일 보냈더라?"`

### 8. 확장-상황구체화
- 구체적 상황/값 포함
- 예: `"합격자한테 메일 나갔는지 확인하고 싶어"`
- 예: `"과락 60점으로 설정하고 싶어"`

### 9. 확장-긴급상황
- 급한 경우
- 패턴: `"~해야 돼"`, `"급하게 ~"`
- 예: `"면접 안내 메일 내용 변경해야 돼"`

## 🎨 질의 작성 원칙

### 1. 자연스러운 발화
- ✅ "접수 기간 일주일 연장할래"
- ❌ "접수 기간 연장 기능 실행 요청"

### 2. 사용자 숙련도 반영
- 기능 이름을 정확히 모르는 상태
- 예: "블라인드 채용 설정" → "학교랑 사진 가리는 거 어디야?"

### 3. 급박한 상황 반영
- "급하게 공고 내려야 돼"
- "지원 마감 1시간 연장해야 해"

### 4. 구체적 상황 포함
- "합격자한테 메일 나갔는지 확인하고 싶어"
- "개발직군 역량검사 설정 바꾸고 싶어"
- "6개월 내 지원자 차단하고 싶어"

### 5. 다양한 표현 방식
- 공손한 요청: "~할 수 있을까요?"
- 직접 명령: "~해줘", "~가자"
- 탐색형: "어디서 ~해?", "~하려면?"
- 축약/구어체: "~할래", "~보자"

## 💡 활용 시 주의사항

### 1. 맥락 처리
- 사용자가 채용명/전형명을 명시하지 않은 경우
- → 시스템이 채용 선택 테이블 제공
- → URL에 파라미터 포함하여 기능에 바로 포커스

### 2. 제외된 항목
- **사용 여부: 아니요** 항목은 확장하지 않음
- 예: CM-05~13 (v14.3.0부터 다른 라우터로 처리)

### 3. 특수 케이스
- **지원서 블라인드**: 비고에 "혼동 주의" 표시
  - 이유: 지원자 접수와 전형 블라인드가 다름
