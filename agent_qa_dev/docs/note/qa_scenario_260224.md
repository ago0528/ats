# 채용에이전트 QA 시나리오

## GNB

[] 좌측 - 홈, 채용 목록, 메시지 현황 페이지 정상 이동
[] '채용 사이트' 이동 가능
[] '평가자 사이트' 이동 가능
[] '응시자 사이트' 이동 가능

## LNB

[] 접고 펼치기 가능

## 홈

1. 'AI 제안' 탭 > 퀵 카드
   - '공채 채용 새성, 수상시 채용 생성, 맞춤 인재 채용 설계' 카드 클릭 시, 플랜 생성 시나리오 (이동 - 적용)
   - '지원 경로별 지원자 현황, 채용 분야별 지원자 현황' 카드 클릭 시, 지원자 관리 시나리오 (통계/분석)
   - 나머지 카드 클릭 시, 로직으로 처리된 이동 에이전트
   - assistantMessage가 정책서와 동일
   - 컨플 정책서: https://midastech.atlassian.net/wiki/spaces/jobplex/database/869630051

2. '채용 인사이트' 탭 > 인사이트 카드
   - 컨플 정책서: https://midastech.atlassian.net/wiki/spaces/jobplex/database/869630051

3. 추천 질문
   - '추천 질문'이 다르게 제공되는 페이지: 공통, 지원자 관리, 플랜 생성
   - 질의 풀 내에서 응답 제공
   - 질의마다 매핑된 에이전트 모드(targetAssistant)가 다름
   - 지라 티켓: https://midasitweb-jira.atlassian.net/browse/NMRS-15809

4. 확장 질의
   - 질의 풀 내에서 응답
   - '건너뛰기' 기능 클릭 시 에이전트가 가장 이상적인 선택제 선택
   - 같은 채팅에서 최근 3개만 제공함
   - 지라 티켓: https://midasitweb-jira.atlassian.net/browse/NMRS-15810

5. 응답 피드백
   - 온/오프 가능하며 Default로 오프 상태
   - 채팅별로 응답 피드백 존재
   - 페이지 재조회해도 저장된 응답 피드백 노출
   - 부정 피드백 클릭한 경우 '너무 느림'과 가은 감정이 담겨있음
   - 지라 티켓: https://midasitweb-jira.atlassian.net/browse/NMRS-14972

6. 위키 에이전트
   - '출처' 태그 옳바르게 나옮 (간헐적 중복 허용)
   - '출처' 태그 클릭 시 본문 페이지 새창 제공

## 프롬프트

1. 채용 생성

```
공개채용으로 세팅해줘
채용명, 공고명을 모두 변경해줘.
채용은 3월부터 시작할 예정이고, 리드타임은 3개월이야.
영업/디자인/마케팅/IT서비스 직군을 채용할 생각이야. 각각 공고를 만들어줘.
채용 전형은 역검 -> 서류 -> 면접 -> 면접(일정조율) -> 면접으로 세팅해줘.
```
