# Backoffice Ideation and Writing

## Overview
Use this skill for planning, concept exploration, and high-clarity writing.

## When to Use
- Defining new features before implementation
- Writing requirement docs, plans, and decision records
- Generating options with explicit trade-offs

## Output Rules
- Start with goal, constraints, and assumptions.
- Provide 2-3 options with recommendation.
- Keep decisions testable and implementation-ready.
- Use concise Korean or English depending on request.
