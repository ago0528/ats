# 배포 준비 체크리스트

ATS 루트 저장소 배포 전 확인 사항입니다.

---

## 1. .gitignore 정비

- [x] Python 캐시/빌드 산출물 제외 (`__pycache__/`, `*.egg-info/`, `dist/`, `build/`)
- [x] 가상환경 디렉터리 제외 (`.venv/`, `venv/`, `ENV/`)
- [x] 환경 변수/비밀 제외 (`.env`, `.env.*`, `!.env.example`)
- [x] 테스트/검증 산출물 제외 (`test_result_*/`, `out.xlsx`, `out_rule_suggestions.json` 등)
- [x] 로그·임시·캐시 제외 (`*.log`, `logs/`, `tmp/`, `.cache/`)
- [x] IDE/OS/Node 관련 제외

배포 시 **절대 커밋하면 안 되는 것**: `.env`, API 키, `*.pem`, `*.key`, 실제 토큰이 들어간 파일.

---

## 2. 환경 설정

### 2.1 Python

- Python 3.8+ (3.10+ 권장)
- 가상환경 사용 권장 → [파이썬_가상환경_설치_가이드.md](./파이썬_가상환경_설치_가이드.md)

### 2.2 agent_qa 앱

```bash
cd agent_qa
pip install -r requirements.txt
cp .env.example .env   # .env에 실제 값 채우기
```

필수 환경 변수 (`.env`):

| 변수 | 용도 |
|------|------|
| `ATS_BEARER_TOKEN` | ATS API 인증 |
| `ATS_CMS_TOKEN` | CMS API |
| `ATS_MRS_SESSION` | MRS 세션 |
| `OPENAI_API_KEY` | 에이전트 평가용 (선택) |
| `OPENAI_MODEL` | 기본: `gpt-5.2` |

## 3. 배포 전 확인

- [ ] 저장소에 `.env` 또는 실제 API 키가 포함된 파일이 없음
- [ ] `agent_qa/.env.example` 존재 여부 확인 (배포 시 참고용)
- [ ] `README.md` 구조/실행 방법이 최신과 일치하는지 확인
- [ ] 필요한 경우 `requirements.txt` 버전 고정 여부 검토

---

## 4. 실행 확인 (로컬)

```bash
# 루트에서
cd agent_qa
source ../.venv/bin/activate   # 또는 venv 경로에 맞게
pip install -r requirements.txt
streamlit run aqb_v1.0.0.py
```

---

## 5. 정리

- **배포 채널**: Git 저장소 클론/풀 시 위 체크리스트와 `.gitignore`를 기준으로 비밀·캐시·테스트 산출물이 제외된 상태로 사용할 수 있습니다.
- **신규 환경**: 클론 후 `agent_qa/.env.example`를 복사해 `.env`를 만들고 값을 채운 뒤 실행하면 됩니다.
